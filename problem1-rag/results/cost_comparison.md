# Cost Comparison: Self-Hosted ChromaDB vs Managed Vector DB

Monthly cost to **store and serve** an index of N vectors, comparing a self-hosted ChromaDB instance against Pinecone (serverless and pod-based). Generated by `results/cost_analysis.py` — edit the constants there to re-run with your own numbers.

## Assumptions (stated explicitly)

- **Embedding dimensionality**: 1536 (text-embedding-3-small), float32 (4 bytes/dim).
- **Metadata**: ~500 bytes/vector (source filename, section, chunk index, etc.).
- **Index overhead**: ~2.0x raw vector bytes for the HNSW graph + working copies (must fit in RAM).
- **Query volume**: 1,000,000 queries/month (a *lightly-queried* index — the scenario where managed always-on pods hurt most).
- **ChromaDB**: runs embedded on one cloud VM (on-demand US prices); vectors held in RAM. Disk over the included amount billed at $0.08/GB-month.
- **Pinecone serverless**: storage $0.33/GB-month + reads $8.25/M read-units (~1 RU/query); steady-state writes excluded.
- **Pinecone pods**: always-on standard pod ~$80/month, ~500,000 vectors/pod at 1536-dim.
- Prices are published US list rates as of Jan 2026 and are approximate; the point is the *shape* of the curve, not the last dollar.

> Note: this table uses the **conservative** 1536-dim assumption. This project's default embeddings are local `all-MiniLM-L6-v2` at **384-dim** — 4x smaller vectors, so storage/RAM (and thus the ChromaDB VM cost) are roughly **4x lower** than shown. The 1536-dim figures are the upper bound.

## Query-volume sensitivity (1M-vector index)

The reads-are-free-after-the-VM property is the crux. For a fixed 1M-vector index (self-hosted VM = **$120/mo flat**, reads included), here is how Pinecone serverless scales with monthly query volume:

| Queries/month | ChromaDB VM (flat) | Pinecone serverless | Cheaper |
| --- | --- | --- | --- |
| 100,000 | $120/mo | $3/mo | serverless |
| 1,000,000 | $120/mo | $10/mo | serverless |
| 10,000,000 | $120/mo | $85/mo | serverless |
| 50,000,000 | $120/mo | $415/mo | ChromaDB |
| 100,000,000 | $120/mo | $827/mo | ChromaDB |

Serverless wins when the index is lightly queried; the fixed-cost VM wins once query volume is high enough that per-query read charges dominate. The crossover is the decision boundary between the two approaches.

## Reading the table

- At every scale, **self-hosted ChromaDB is a fixed VM cost** that grows in big steps only when the index no longer fits in RAM. It does not scale with query volume.
- **Pinecone pods** are the classic "always-on pods" cost: you pay per pod per hour whether or not you query, and pod count grows linearly with vectors — which is exactly the failure mode a large, lightly-queried index hits.
- **Pinecone serverless** removes the always-on premium and is competitive at small scale, but its storage + per-query read charges rise with both index size and query volume, so a high-QPS or very large index can overtake a single VM.
- **Trade-offs we accept for the ChromaDB cost win**: we self-manage the VM (patching, backups, monitoring), we get no built-in HA/replication or autoscaling, and a single-node index is bounded by one machine's RAM. Those are real operational costs the managed price is partly buying.
