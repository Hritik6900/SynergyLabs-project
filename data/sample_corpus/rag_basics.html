Retrieval-Augmented Generation (RAG)

The core idea

Retrieval-augmented generation grounds a language model's answer in retrieved documents instead of relying only on the model's parametric memory. At query time, the system embeds the question, retrieves the most similar chunks from a vector store, and passes those chunks to the model as context. This reduces hallucination and lets the system answer questions about private or recent data the model was never trained on.

Chunking

Documents are split into chunks before embedding. Chunk size trades off precision against context: small chunks retrieve very specific passages but may lose surrounding context, while large chunks preserve context but dilute the embedding. A common default is a few hundred tokens with a small overlap so that information spanning a boundary is not lost.

Grounding and citations

A well-behaved RAG system cites the chunks it used so answers are auditable. If no retrieved chunk is sufficiently similar to the question, the system should say it has no relevant context rather than fabricate an answer.

Evaluation

RAG quality is measured on two layers. Retrieval quality uses information-retrieval metrics such as Recall@k, Mean Reciprocal Rank (MRR), and normalized Discounted Cumulative Gain (nDCG). Answer quality is measured for faithfulness (is the answer supported by the retrieved context) and relevance (does it address the question), often using an LLM as a judge.
